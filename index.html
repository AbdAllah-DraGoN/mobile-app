مقدمة في React
الدرس 1

React هي مكتبة JavaScript مفتوحة المصدر لبناء واجهات المستخدم. تم تطويرها بواسطة فيسبوك في عام 2013، وأصبحت من أكثر المكتبات استخداماً في تطوير تطبيقات الويب الحديثة.

تتميز React بنهجها الفريد في بناء واجهات المستخدم باستخدام مفهوم المكونات والـ Virtual DOM، مما يجعل التطبيقات أسرع وأكثر كفاءة.

المكونات في React
الدرس 2

المكونات (Components) هي وحدات البناء الأساسية في React. كل مكون له مسؤولية محددة ويمكن إعادة استخدامه في أجزاء مختلفة من التطبيق.

يمكن تقسيم المكونات في React إلى نوعين أساسيين: المكونات الوظيفية (Functional Components) والمكونات الصنفية (Class Components). في التطبيقات الحديثة، أصبحت المكونات الوظيفية هي الخيار المفضل بسبب بساطتها وقوتها مع استخدام Hooks.

قابلة لإعادة الاستخدام ولها مسؤولية محددة
لا يمكن تخصيصها بعد إنشائها
JSX وطريقة الكتابة
الدرس 3

JSX (JavaScript XML) هو امتداد لبناء الجملة في JavaScript يشبه HTML. يسمح لك بكتابة عناصر HTML مباشرة داخل ملفات JavaScript، مما يجعل كتابة واجهات المستخدم أكثر سهولة وبديهية.

باستخدام JSX، يمكنك دمج عناصر HTML مع منطق JavaScript في نفس الملف. يتم تحويل JSX إلى JavaScript قياسي بواسطة مترجمات مثل Babel قبل تشغيل التطبيق في المتصفح.

الحالة (State) والخصائص (Props)
الدرس 4

الحالة (State) والخصائص (Props) هما مفهومان أساسيان في React لإدارة البيانات داخل التطبيق.

Props هي القيم التي تُمرر من المكون الأب إلى المكون الابن، وهي للقراءة فقط ولا يمكن تعديلها داخل المكون الذي يستقبلها.

State هي البيانات التي يديرها المكون داخلياً ويمكن تغييرها من خلال دوال محددة، مما يؤدي إلى إعادة عرض المكون عند تغيير الحالة.

ما الفرق بين State و Props؟

لا يوجد فرق بينهما
Props تمرر من الخارج، State تدار داخلياً
Props و State كلاهما ثابت ولا يمكن تغييره
دورة حياة المكون وال-Hooks
الدرس 5

تمر مكونات React بدورة حياة (Lifecycle) من التركيب والتحديث وإزالة التركيب. في المكونات الصنفية، يمكن التفاعل مع دورة الحياة من خلال طرق مثل componentDidMount و componentDidUpdate.

مع إدخال Hooks في React 16.8، أصبح من الممكن استخدام ميزات مثل الحالة ودورة الحياة في المكونات الوظيفية. أهم Hooks هي:

useState: لإدارة الحالة المحلية
useEffect: للتعامل مع الآثار الجانبية مثل جلب البيانات ودورة الحياة
useContext: للوصول إلى بيانات السياق
